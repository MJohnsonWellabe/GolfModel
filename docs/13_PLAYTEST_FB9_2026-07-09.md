## What shipped this session

### Putting rework (the headline)
- **Off-green putts reach their target.** The putter launch speed used to assume
  green friction always; it now uses the friction the putt actually rolls
  through (path-averaged), so a putt from fringe/rough carries its intended
  distance. Pure-green putts average to green friction, so make-rates are
  unchanged (Appendix-A sim still passes).
- **3-foot gimmes.** Anything inside 3 ft is conceded automatically — counted as
  a holed tap-in, no putt required (`tryGimme`).
- **The flag is pulled while putting** (`setPinPulled`) so a putt can't hit the
  flagstick.
- **Honest cup.** The visual cup + open-cup ring were drawn larger than the
  physics capture radius, so balls "rolled over the hole" that never actually
  reached it. Shrunk the *visual* to match the (unchanged) capture — no widening,
  keeps putting appropriately hard.
- **De-compressed putt camera** (~17° vs ~6°) so a 30 ft putt reads as 30 ft.

### Cameras, aim, shot shape
- Aim readout clamps to the screen edge instead of vanishing when the pin
  projects off-view (it was mistaken for "only shows when you won the hole").
- Shot shape now moves the aim dots (the strike-drag handler recomputes the
  preview) **and** visibly bends flight (stronger side-spin).
- The intro flyover is a real tee → mid-fairway → green travel again.
- The ball-follow camera looks at the ball (not 26 units ahead) and no longer
  jumps to the landing zone mid-flight.

### Courses
- **Trees are per-trunk now.** Flight collision tests individual tree canopies,
  not the whole tree polygon — a ball threading a gap flies on; only a ball that
  truly reaches a tree stops. Rising balls are stopped too (not just descending).
  The AI punches *out* of trees toward open ground. Pine Alley's centre tree was
  offset to leave a fair line (an unavoidable centre block broke the Course
  Bible). Regression tests cover rising-strike + gap-threading.
- **Sable Bay water comes into play:** the island green is smaller (rx 82→60,
  plays ~+0.4 for a strong golfer), the par 4 has ocean down its whole left side
  plus a creek across the approach, and the par 5 has a forced water carry.

### Content
- **Full 25-character roster.** The other 15 chibis from the Cute Characters 4
  pack are wired in. Character select shows the *whole* roster — locked ones get
  a lock + price badge and open the Store when tapped. 5 free, 20 unlockable. New
  `npm run portraits` renders all 25 portraits consistently (dev harness).
- **Ace Challenge is a game mode** (after Scramble), and its Course step picks
  which course's par 3 to tee off — all three are now available.
- **Google linking is on the main menu** (🔗 Link Google), not buried in Profile.
- **Apparel customization** (see the constraint note below): outfit colorways +
  club skins, equippable like ball/trail tints.
- **Tournament history** ("My Tournaments") persists on the profile and lists
  past events to reopen.

## Vision gap review — where we stand vs the docs

Cross-referenced against `00_MASTER_VISION`, `02_GAME_DESIGN_DOCUMENT`,
`05_DEVELOPMENT_ROADMAP`, `09_PRODUCT_REQUIREMENTS`, `10_COURSE_DESIGN_BIBLE`.

**Met / now met.** All nine roadmap phases are built. This session closed the
parts of Phase 1B (camera), Phase 2 (putting & balance), and Phase 4 (shot
shaping) that were built but failed the playtest bar. The PRD gameplay checklist
(swing, wind, aiming, physics, collision, hazard penalties, putting, chipping,
shot shaping, spin, dynamic cameras) is functional; three polished courses with
distinct identities exist; progression, store, accounts, tournaments are live.

**Gaps still open:**

1. **Per-garment apparel is not achievable with the current assets.** The vision
   (Phase 7) lists shirts / dresses / shoes / hats / club skins as separate
   items. The chibi models ship **one clothing material** (`characters`) for the
   whole outfit plus a skin material — there are no separable garment meshes to
   recolor independently. We shipped what the mesh supports: a **whole-kit outfit
   colorway** (albedo multiply) + a **club-skin** tint (the club is a separate
   procedural mesh). True per-garment customization would need either
   garment-split character models or a texture-swap atlas per character — a
   content/pipeline project, not a code tweak.

2. **Named golfer identities (Jeff / Matt / Zac).** `00_MASTER_VISION` names
   three golfers with specific power/accuracy/short-game identities and target
   ratings. The v2 pivot represents playstyle via **archetypes**
   (`src/data/archetypes.ts`) decoupled from the (generic-name) chibi avatars, so
   the specific Jeff/Matt/Zac identities aren't surfaced by name. If those
   personas matter, a small pass could name three archetypes accordingly and/or
   pin default avatars to them.

3. **Player-attribute spread.** The vision wants a dramatic 250–320 yd driver
   spread and "immediately different" golfers. Appendix-A driver distances are
   honored by the sim; whether the *felt* spread meets the vision is a tuning
   question worth a dedicated balance pass (it wasn't in this playtest's list).

4. **Accounts need three Firebase-console steps** by Matt to work end-to-end
   (`docs/FIREBASE_SETUP.md`): enable Anonymous + Google sign-in, add the
   authorized domains, publish the DB rules. The code + the new prominent
   main-menu link are ready.

5. **On-device fps testing** — headless measures render cost, not true fps; the
   `docs/DEVICE_MATRIX.md` rows still need real phones.

6. **Deferred tech debt:** `src/slice3d/main.ts` is the ~2.6k-line composition
   root; still the top refactor item, kept behind the Playwright net.

**Post-1.0 (explicitly future):** weather, career mode, friends, replay,
practice range, spectator, live multiplayer, more courses.
